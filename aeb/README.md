# 1/32 스케일 실제차형 AEB

## 프로젝트 핵심

RC카에 실제 차량 AEB와 비슷한 판단 구조를 넣은 긴급 제동 시스템이다.

단순히 거리가 가까우면 멈추는 방식이 아니라,

```text
객체 인식 + 거리 + 상대속도 + 예상 경로 + TTC + 필요 감속도
```

를 함께 계산해서 제동 여부를 판단한다.

## 사용 센서

| 센서 | 역할 |
| --- | --- |
| 전방 ToF | 정면 장애물 거리 측정 |
| 좌측 초음파 | 좌측 전방 대각선 장애물 감지 |
| 우측 초음파 | 우측 전방 대각선 장애물 감지 |
| USB 카메라 | 객체 종류와 위치 인식 |

## 전체 흐름

```text
센서값 수집
-> 객체 인식
-> 왼쪽 / 앞 / 오른쪽 위치 판단
-> 센서 거리 융합
-> 1/32 실제차 환산
-> TTC / 필요 감속도 / 정지거리 계산
-> AEB 상태 결정
-> 경고 또는 제동
```

## 1/32 스케일 환산

RC카에서 측정한 거리와 속도를 실제차 기준으로 환산해서 판단한다.

```text
실제차 환산 거리 = RC 거리 * 32
실제차 환산 속도 = RC 속도 * 32
RC 정지거리 = 실제차 정지거리 / 32
```

예시:

```text
Speed : 32.0 km/h real-eq
```

뜻:

```text
실제차 32km/h 상황을 1/32 크기로 축소해서 실험 중
```

## 판단 기준

### TTC

```text
TTC = 장애물 거리 / 제동으로 줄일 수 있는 접근속도
```

| 단계 | 기준 |
| --- | --- |
| 경고 | TTC <= 2.00초 |
| 부분 제동 | TTC <= 1.20초 |
| 긴급 제동 | TTC <= 0.60초 |

### 필요 감속도

| 단계 | 기준 |
| --- | --- |
| 경고 | 2.0 m/s^2 이상 |
| 제동 준비 | 3.0 m/s^2 이상 |
| 부분 제동 | 4.5 m/s^2 이상 |
| 긴급 제동 | 7.0 m/s^2 이상 |

### 정지거리

```text
정지거리 =
속도 * 시스템 지연시간
+ 속도^2 / (2 * 최대 감속도)
+ 안전 여유거리
```

현재 값:

```text
최대 감속도 = 7.85 m/s^2
시스템 지연 = 0.35초
안전 여유 = 실제차 기준 1.0m
```

## 중요한 로직

내 차가 정지 중이면 장애물이 가까워져도 AEB 제동을 걸지 않는다.

이유:

```text
AEB는 내 차의 움직임 때문에 생기는 충돌 위험을 줄이는 장치이기 때문
```

그래서 접근속도는 이렇게 계산한다.

```text
제동용 접근속도 = min(센서 접근속도, 내 차 속도)
```

## AEB 상태

| 상태 | 의미 |
| --- | --- |
| MONITORING | 정상 감시 |
| FCW | 충돌 경고 |
| BRAKE_PREFILL | 제동 준비 |
| PARTIAL_BRAKE | 부분 제동 |
| FULL_BRAKE | 긴급 제동 |
| STOP_HOLD | 정지 유지 |
| DEGRADED | 일부 센서 제한 |
| SENSOR_FAULT | 센서 오류 |
| RELEASE | 제동 해제 |

## 객체 인식

카메라는 다음 객체를 인식한다.

```text
person, bicycle, car, motorcycle, bus, truck
```

객체 위치는 다음처럼 나눈다.

```text
주행 경로 안 -> 앞
x좌표 음수 -> 왼쪽
x좌표 양수 -> 오른쪽
```

## 센서 융합

```text
전방 중앙 위험 = 카메라 + ToF
좌우 근접 위험 = 카메라 + 좌우 초음파
카메라 실패 시 = ToF/초음파 거리 기반 판단
ToF 타겟 없음 = 고장이 아니라 전방 유효 장애물 없음
```

## 발표용 한 문장

이 AEB는 거리 하나로 멈추는 단순 제동 장치가 아니라,
객체 종류, 위치, 거리, 자차 속도, TTC, 필요 감속도, 정지거리까지 계산해서
실제 차량 AEB와 유사한 판단 흐름을 1/32 RC카 스케일로 구현한 시스템이다.

## 실행

```bash
python3 aeb.py
```

